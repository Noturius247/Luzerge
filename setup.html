Step-by-step instructions for setting up your domain with Luzerge. Choose your plan type below to see the guide tailored for you.

Quick Overview

What you'll accomplish in 6 steps

1

Create Cloudflare Account

Sign up at cloudflare.com. A free account is enough — Luzerge adds the advanced features on top.

2

Add Domain to Cloudflare

Cloudflare imports your existing DNS records. Enable the proxy (orange cloud) on your web records.

3

Update Nameservers

Replace your domain registrar's nameservers with the 2 Cloudflare nameservers. Takes 5 min–48 hrs.

4

Configure Cloudflare Settings

Enable SSL "Full" mode, Always Use HTTPS, and verify your proxy records are orange.

5

Get API Credentials

Create an API token and copy your Zone ID from Cloudflare so Luzerge can manage your domain.

6

Submit Domain + Enable Monitoring

Add your domain in the Luzerge dashboard, then turn on uptime checks, alerts, and all monitoring tools.

1

Create a Cloudflare Account

Free account · 2 minutes

1

Go to cloudflare.com and sign up

Click Sign Up, enter your email address and a password. Cloudflare's free plan is all you need.

2

Verify your email

Check your inbox for a verification email from Cloudflare and click the confirmation link to activate your account.

3

Select the Free plan

When prompted, choose Free. Luzerge provides all advanced caching, DDoS protection, WAF, and monitoring on top of Cloudflare's free infrastructure — you don't need a paid Cloudflare plan.

✓
One Cloudflare account handles all your domains. You don't need a separate account per domain.
2

Add Your Domain to Cloudflare

Cloudflare auto-imports your existing DNS records

1

Click "Add a site"

In the Cloudflare dashboard, click + Add a site. Enter your root domain without www or https — e.g. example.com.

2

Select the Free plan

Scroll to the bottom of the plan selection page and choose Free, then click Continue.

3

Review imported DNS records

Cloudflare automatically scans and imports your existing DNS records. Review them carefully — confirm all A records (your server IP), CNAME records, and MX records (for email) are present.

!
Do not delete any records Cloudflare imported. Missing records can break your website or email.
4

Enable proxy (orange cloud) on web records

For every A or CNAME record that points to your website, click the cloud icon to make it orange (Proxied). This enables the CDN, DDoS protection, and all Luzerge monitoring features. Keep email (MX) records as grey (DNS only).

A record: @ → YOUR.SERVER.IP.HERE → Proxied ☁ (orange) A record: www → YOUR.SERVER.IP.HERE → Proxied ☁ (orange) MX record: @ → mail.example.com → DNS only (grey — required for email) TXT record: @ → v=spf1 ... → DNS only (grey)
3

Update Your Nameservers

Routes your domain through Cloudflare · Takes 5 min – 48 hours

1

Copy Cloudflare's 2 nameservers

After adding your site, Cloudflare shows you 2 nameservers assigned to your account. Copy both — they look like this (yours will be different):

ada.ns.cloudflare.com bob.ns.cloudflare.com
2

Log in to your domain registrar

Go to where you bought your domain and find the Nameservers or DNS Settings section:

GoDaddyMy Domains → Manage DNS → Nameservers → Change
NamecheapDomain List → Manage → Nameservers → Custom DNS
Google DomainsDNS → Custom nameservers
HostingerDomains → Manage → DNS / Nameservers
BluehostDomains → My Domains → Manage → Nameservers
Other RegistrarsSearch for "Nameservers" or "DNS" in your registrar's control panel
3

Replace existing nameservers

Delete all existing nameservers and paste in the 2 Cloudflare ones. Save the changes. Most registrars take 10–30 minutes; the maximum is 48 hours.

4

Verify in Cloudflare

Back in the Cloudflare dashboard, click Done, check nameservers. Cloudflare sends you an email when your domain is active. The domain status will change from Pending to Active.

✓
Check propagation at dnschecker.org — search your domain's NS records to see if Cloudflare nameservers are showing globally yet.
4

Required Cloudflare Settings

Configure these after nameservers are active

Once your domain is active in Cloudflare, configure these 6 settings so Luzerge can monitor and manage your domain correctly. These are the most important settings — skipping any of them can cause issues.

SSL/TLS → Overview
Encryption Mode
Set to: Full
Use Full if you have a self-signed cert on your server, or Full (Strict) if you have a valid CA-issued cert. Never use Flexible — it causes infinite redirect loops on most servers.
SSL/TLS → Edge Certificates
Always Use HTTPS
Turn ON
Redirects all HTTP traffic to HTTPS automatically. Required for Luzerge's SSL monitoring to work correctly and for modern browser compatibility.
SSL/TLS → Edge Certificates
Minimum TLS Version
Set to: TLS 1.2
Blocks old insecure TLS versions (1.0, 1.1). TLS 1.2 is compatible with all modern browsers and is the security standard.
DNS → Records
Proxy Status on A/CNAME
Orange Cloud (Proxied)
Your web records must be proxied — this routes traffic through Cloudflare enabling CDN, DDoS protection, WAF, and analytics. Grey (DNS only) records bypass everything.
Speed → Optimization → Content Optimization
Auto Minify
Enable (optional)
Minifies HTML, CSS, and JavaScript automatically. You can also control this from the Luzerge dashboard's Minification panel, which syncs with Cloudflare.
Caching → Configuration
Caching Level
Keep: Standard
Leave at Standard. Luzerge controls cache purging from the dashboard — you don't need to change this manually. Changing it can conflict with purge operations.
!
Critical: Never use "Flexible" SSL mode Flexible mode encrypts only the connection between the visitor and Cloudflare, not between Cloudflare and your server. This causes infinite redirect loops on most web servers and is a security risk.
5

Get API Credentials

Create an API token and find your Zone ID

Provide API credentials so Luzerge can perform cache purges, read analytics, and manage DNS on your behalf. Your token is stored encrypted — it's never visible in plain text after submission.

Part A — Create an API Token

1

Open API Tokens in your Cloudflare Profile

Click your account icon (top-right corner of Cloudflare) → Profile → API Tokens tab.

2

Create a Custom Token

Click Create Token, scroll past the templates, and click Get started under Create Custom Token. Give it a name like Luzerge.

3

Add required permissions

Click Add permission and add each of these:

Zone → Zone → Read Zone → Zone Settings → Read Zone → DNS → Read Zone → SSL and Certificates → Read Zone → Firewall Services → Read Zone → Cache Purge → Purge Account → Analytics → Read

Under Zone Resources, select Specific zone and choose your domain from the dropdown.

4

Copy the token immediately

Click Create Token. Copy the long string that appears — it's shown only once. If you close the page without copying, delete the token and create a new one.

!
Store this token securely. Anyone with this token can manage your Cloudflare zone.
i
API Token Summary

After creation, your token should show the following on the API Tokens page:

Luzerge Dashboard

luzerge.com - Zone:Read, Zone Settings:Read, DNS:Read, SSL and Certificates:Read, Firewall Services:Read, Cache Purge:Purge, Analytics:Read

Part B — Find Your Zone ID

1

Open your domain's overview in Cloudflare

Click on your domain name in the Cloudflare dashboard to open its main overview page.

2

Copy the Zone ID from the right sidebar

Scroll down the right column. You'll see Zone ID — a 32-character hexadecimal string. Click Copy.

Zone ID example: a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4
6

Submit Your Domain to Luzerge

You're almost done

1

Sign in to Luzerge

Go to luzerge.com and click Dashboard. Sign in with your Google account.

2

Click "Add Domain" on the Overview panel

On the Overview panel, click the Add Domain button. Enter your domain name without www — e.g. example.com.

3

Provide your API credentials

Paste your Cloudflare API Token and Zone ID from Step 5 when prompted. Your token is stored encrypted — it's never visible in plain text after submission.

4

Wait for activation

Domain status shows Pending initially. Activation is automatic once your credentials validate — usually within a few seconds.

5

Enable monitoring features

Once the domain is Active, go to Settings → Notifications to set up alerts, and Security → Uptime Monitoring to enable uptime checks. See the feature guide below for full details.

✓
Once active, all dashboard panels become available — Analytics, SSL, DNS, WAF, DDoS, Uptime, Image Optimization, and Minification are all ready to use.

Dashboard Features Guide

How to use every monitoring and management tool

Uptime Monitoring

All Plans

Luzerge checks your website every 1–5 minutes from multiple locations. If your site goes down, you receive an instant email alert. When it recovers, you get another alert with the total downtime duration.

→

Enable uptime checks

Go to Security → Uptime Monitoring in the sidebar. Toggle Enable uptime checks for your domain. Choose your check interval: 1 minute (most sensitive), 2 minutes, or 5 minutes.

→

Set up downtime alerts

Go to Settings → Notifications. Enable Downtime alerts and Recovery alerts. Confirm your alert email — you can use a different email than your login (e.g. a team inbox).

→

Read your uptime history

The Uptime panel shows your overall uptime percentage, average response latency, a timeline of recent checks, and a list of past downtime incidents with duration and cause.

SSL / TLS Monitoring

All Plans

Luzerge tracks your SSL certificate expiry and sends email warnings 30 days before it expires. The dashboard shows your certificate issuer, expiry date, TLS version, and encryption mode.

→

View SSL certificate details

Go to Network → SSL/TLS in the sidebar. See your certificate issuer (e.g. Let's Encrypt, DigiCert), exact expiry date, and current TLS version.

→

Enable SSL expiry alerts

Go to Settings → Notifications and turn on SSL expiry alerts. You'll get an email 30 days before expiry — enough time to renew before any outage.

✓
Cloudflare provides free auto-renewing SSL certificates. As long as your domain is proxied through Cloudflare (orange cloud), your SSL certificate renews automatically — you never need to manually renew.

CDN & Cache Purge

All Plans

Luzerge uses Cloudflare's global CDN (300+ edge locations) to cache your website close to your visitors, dramatically reducing load times. You control cache purges directly from the dashboard — no need to log into Cloudflare.

→

Manual cache purge

In the Overview panel, click the Purge Cache button next to your domain. This instantly clears all cached files at every Cloudflare edge location worldwide, forcing fresh content to be fetched from your server. Use this after deploying updates.

→

Auto-purge schedule

Go to Settings → Domain Defaults. Enable Auto-purge and choose an interval: 1 hour, 6 hours, 12 hours, or 24 hours. Useful for sites that update frequently (blogs, e-commerce, news).

→

Purge history

All cache purge events are logged. View the history in Performance → Analytics — each entry shows the timestamp, purge type (manual or auto), and success/failure status.

DNS Management

All Plans

View, add, and edit your domain's DNS records directly from the Luzerge dashboard without needing to log into Cloudflare separately. Changes are applied in real-time via the Cloudflare API.

→

View all DNS records

Go to Network → DNS Management. See all A, AAAA, CNAME, MX, TXT, NS, SRV, and CAA records with their current values, TTL, and proxy status.

→

Add or modify records

Click Add Record to create new DNS entries. Click any existing record to edit its value, TTL, or proxy status. Common use cases: adding a CNAME for a subdomain, adding TXT records for domain verification, or updating your server IP.

ℹ
DNS changes through Luzerge take effect within seconds globally. You don't need to wait for propagation — Cloudflare's API updates records instantly.

WAF / Firewall

Managed Plans

The Web Application Firewall (WAF) analyzes and filters HTTP traffic to block malicious requests before they reach your server — SQL injection, cross-site scripting (XSS), path traversal attacks, bad bots, and more. Cloudflare's managed WAF rules are automatically updated as new threats emerge.

→

View WAF statistics

Go to Security → WAF/Firewall. See the total blocked requests count, top threat types (SQLi, XSS, bots), top attacking IPs, and a timeline of blocked events over the last 24h/7d.

→

Create custom firewall rules

Click Add Rule to create custom rules that allow or block traffic based on: IP address, IP range, country, ASN, user-agent string, URL path, or request headers.

Example rules: Block all traffic from a specific country · Whitelist only your admin IP · Block requests containing specific strings in the URL.

→

Bot protection

Luzerge configures Cloudflare's Bot Fight Mode which automatically challenges known bad bots (scrapers, credential stuffers, vulnerability scanners). View blocked bot traffic in the WAF panel.

DDoS Protection

All Plans

DDoS protection is always-on and automatic through Cloudflare's global network. Luzerge monitors attack events and provides L3/L4 (network layer) and L7 (application layer) protection. Your server IP stays hidden behind Cloudflare at all times.

→

View attack statistics

Go to Security → DDoS Protection. See protection status for L3/L4 and L7, attack event history, peak traffic during attacks, and how much traffic was blocked vs allowed.

→

Under Attack Mode

If you're experiencing an active attack and seeing slowness, click Enable Under Attack Mode from the DDoS panel. This adds an interstitial JavaScript challenge for all new visitors, dramatically reducing bot traffic while real users pass through in ~5 seconds.

ℹ
Never expose your real server IP address in DNS — always keep your A records proxied (orange cloud) through Cloudflare. If your origin IP leaks, attackers can bypass Cloudflare and attack your server directly.

Analytics

All Plans

Real-time traffic analytics powered by Cloudflare — captured at the network edge, not with JavaScript tracking scripts. This means it's faster, more accurate, doesn't affect page performance, and captures traffic that would normally be blocked by ad blockers.

→

Traffic overview

Go to Performance → Analytics. The overview shows: total requests, bandwidth used, unique visitors, cache hit rate, and response status codes (2xx success / 4xx client errors / 5xx server errors). Toggle between 24h, 7 days, and 30 days views.

→

Cache performance

The cache hit rate tells you what percentage of requests are served from Cloudflare's cache vs. hitting your origin server. A higher cache hit rate means faster load times and less server load. Aim for 70%+ for content-heavy sites.

→

Geographic traffic breakdown

See which countries your visitors are coming from and how much bandwidth each country uses. Useful for identifying your primary audience and optimizing for their region.

Image Optimization & Minification

All Plans

Speed up your website automatically at the network level — no code changes needed.

→

Image Optimization (Polish + WebP)

Go to Performance → Image Optimization. Enable Cloudflare Polish to automatically compress JPEG and PNG images on-the-fly with no visible quality loss (lossless by default). Enable WebP conversion to serve WebP format to browsers that support it — WebP is 25–34% smaller than JPEG at the same quality.

→

Minification (HTML/CSS/JS)

Go to Performance → Minification. Enable auto-minification for HTML, CSS, and JavaScript files. This strips whitespace, comments, and unnecessary characters from your code files before serving them to visitors — typically reduces file sizes by 10–30%.

✓
After enabling optimization features, purge your cache so visitors immediately receive the optimized versions of your files.

Troubleshooting

Common issues and how to fix them

Domain still showing "Pending" after 24 hours
First verify your Cloudflare nameservers have fully propagated at dnschecker.org. If they show correctly globally, email luzergeservices@gmail.com with your domain name and we'll check the status manually.
Website shows ERR_TOO_MANY_REDIRECTS or SSL error
This almost always means Cloudflare SSL mode is set to Flexible. Go to Cloudflare → SSL/TLS → Overview and change to Full or Full (Strict). Also make sure "Always Use HTTPS" is enabled. Clear your browser cache after changing.
Nameservers updated but domain is still "Pending" in Cloudflare
Propagation can take up to 48 hours. Check progress at dnschecker.org — search for your domain's NS records. Once Cloudflare nameservers are showing globally, return to Cloudflare and click Re-check nameservers or wait — Cloudflare checks automatically every few hours.
"Invalid credentials" error when submitting domain (Free plan)
Your API token may be missing required permissions, expired, or was accidentally revoked. Go to Cloudflare → API Tokens, delete the existing Luzerge token, and create a new one following Step 5 exactly. Make sure to include all 6 permission entries and set Zone Resources to your specific domain.
Cache purge not working or old content still showing
First make sure your DNS records are proxied (orange cloud) — DNS-only records bypass the CDN entirely. Click Purge All in the Overview panel. If content is still stale, also clear your browser's local cache (Ctrl+Shift+R or Cmd+Shift+R). For API errors, verify your API token still has Cache Purge permission.
Not receiving downtime alert emails
Check Settings → Notifications and confirm downtime alerts are enabled with the correct email address. Check your spam/junk folder — add luzergeservices@gmail.com to your contacts to prevent filtering. Also confirm that uptime monitoring is enabled in Security → Uptime Monitoring.
Analytics showing no data or zero traffic
Analytics require your domain to be Active and DNS records to be proxied (orange cloud). Data starts appearing within a few hours of first proxied traffic. If it's been over 24 hours with active traffic and no data, contact support with your domain name.
Email stopped working after switching nameservers to Cloudflare
Cloudflare may not have imported your MX records, or they may have been accidentally proxied (they must be DNS-only / grey cloud). Go to Cloudflare → DNS → Records, verify all MX records are present and set to DNS-only (grey cloud). Also check SPF and DKIM TXT records.
ℹ
Need more help? Email luzergeservices@gmail.com with your domain name, your plan type, and a description of the issue. Include any error messages you're seeing. We respond within 24 hours. You can also ask Lumi (the chat icon in the bottom-right corner) for quick guidance.
