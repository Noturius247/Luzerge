Security

We ask you to trust us with your Cloudflare API credentials. Here's exactly how we protect them and the rest of your data.

1. Infrastructure

Luzerge is built on battle-tested infrastructure:

  • Database — Supabase PostgreSQL in the Singapore region, with automatic backups
  • Edge delivery — Cloudflare CDN with 300+ global edge locations
  • API — Supabase Edge Functions running on Deno Deploy
  • CI/CD — GitHub Actions for automated deployments
  • Transport — All communication encrypted via HTTPS / TLS 1.2+

2. Authentication

We use Google OAuth 2.0 exclusively for sign-in. This means:

  • We never see or store your password
  • We only request your basic profile (name, email, profile picture)
  • We do not access your Google Drive, Gmail, or any other Google services
  • Session tokens are managed by Supabase Auth and stored in your browser's local storage
  • No third-party tracking cookies are used

3. Data Encryption

Your Cloudflare API tokens are encrypted at rest in our database. They are only decrypted server-side when performing actions you request (cache purges, analytics queries, etc.) and are never exposed in API responses or logs.

  • All data in transit is encrypted via TLS 1.2+
  • Database connections use SSL
  • API tokens are encrypted before storage — even database administrators cannot read them in plain text

4. Access Control

We enforce strict data isolation using Supabase Row Level Security (RLS) at the database level. This means:

  • You can only read and modify your own domains and tokens
  • RLS policies are enforced by PostgreSQL itself — not application code
  • Admin access is restricted to the platform operator

5. What We Store

  • Your name, email, and profile picture (from Google)
  • Domain names you add to your account
  • Cloudflare Zone IDs and API tokens (encrypted)
  • Cache purge action logs
  • Monitoring and analytics data for your domains

6. What We Do NOT Store

  • Your Google password
  • Full Cloudflare account credentials — only scoped API tokens with the permissions you choose
  • Payment card numbers — payments are handled via GCash, Maya, bank transfer, or PayPal
  • Browsing history or personal data beyond what is listed above

7. Incident Response

In the unlikely event of a data breach:

  • Affected users will be notified via email within 72 hours
  • We will immediately rotate any potentially compromised encryption keys
  • A public incident report will be posted on our status page

8. Responsible Disclosure

If you discover a security vulnerability, please report it to luzergeservices@gmail.com. We take all reports seriously and will respond within 48 hours.

9. Contact

For security questions or concerns, contact us at luzergeservices@gmail.com
